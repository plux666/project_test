# Тестовый Гусь

Про Tailwind поздно вспомнил, поэтому стилизация на scss с намеком на БЭМ. MUI в качестве UI библиотеки. Выбрана
что бы протестировать как она ощущается в 2025 (вроде неплохо). 

Не стал реализовывать сложную логику обработки тапов т.к.
не смог быстро выдумать "непробиваемую" стратегию. "Оптимистичный" подход мне, как игроку в контр-страйк, не нравится
(`What you see is not what you get`). 
Возможно мой вариант еще не плох, потому что гарантирует актуальность данных на каждый отрезок времени.

Не сразу обратил внимание, что в списке раундов должны быть только "активные" и "будущие" раунды. По какой то причине
бэкенд не может в одном запросе прислать раунды по двум статусам, поэтому код не очень опрятный. Изначально делал 
список с кнопкой "Загрузить еще" (она работает, но быстро теряет актуальность). На бесконечный скролл не 
хватило времени. Было бы полезно придумать механику, которая синхронизирует скоротечные курсоры (если у нас раундов
на 2 страницы по 10, то через минуту их станет сильно меньше и "вторая" страница перестанет существовать).

Так же заметил, что бэкенд перестает принимать тапы на пару секунд раньше. Времени не хватило, по хорошему нужна обработка
ошибки с принудительным переключением статуса. В идеале указание статуса с бэкенда.
